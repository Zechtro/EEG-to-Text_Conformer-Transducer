# 📊 VISUALISASI TRAINING & RESULTS

## Overview

Setelah training selesai dengan `all0.py`, akan di-generate **3 type visualisasi** otomatis:

1. **training_history.png** - Dibuat otomatis oleh `all0.py`
2. **training_analysis.png** - Generate dengan `analyze_results.py`
3. **test_results_analysis.png** - Generate dengan `analyze_results.py`

---

## 📈 Cara Generate Visualisasi

### Otomatis (Saat Training)
```bash
python src/pipelines/training/all0.py
```

Akan menghasilkan:
- `training_history.png` ✓ (otomatis)
- Semua output lainnya

### Manual/Analisis Ulang
```bash
python src/pipelines/training/analyze_results.py
```

Akan menghasilkan:
- `training_analysis.png` ✓
- `test_results_analysis.png` ✓

## 📌 Deskripsi Setiap Visualisasi

### 1️⃣ training_history.png (Basic)

**Generated by:** `all0.py` (otomatis)

**Isi:**
- **Top:** Training Loss vs Validation Loss
  - Blue line = Train Loss (mengecil → learning baik)
  - Red line = Val Loss (mengecil → generalisasi baik)
  - Gap besar = indikasi overfitting

- **Bottom:** Validation CER per Epoch
  - Green line = CER metrics (mengecil = improvement)
  - Lower is better

**Cara membaca:**
```
Ideal:
├─ Train loss turun ➘
├─ Val loss turun ➘ (hampir selaras dengan train)
└─ Val CER turun ➘

Overfitting Warning:
├─ Train loss terus turun ➘
└─ Val loss mulai naik ➚

Underfitting Warning:
├─ Both losses masih tinggi
└─ CER tidak banyak berkurang
```

---

### 2️⃣ training_analysis.png (Detailed)

**Generated by:** `analyze_results.py`

**Isi (6 subplots):**

1. **Top Panel: Train vs Val Loss**
   - Filled area menunjukkan gap antara train dan val
   - Semakin kecil gap = semakin baik generalisasi

2. **Middle-Left: Val CER Trend**
   - Direct metric untuk model performance
   - Target: CER < 0.2 dianggap excellent

3. **Middle-Right: Loss pada Skala Logaritmik**
   - Membantu visualisasi konvergensi
   - Slope gradual = learning rate mungkin terlalu kecil
   - Slope steep = learning rate mungkin bagus

4. **Bottom-Left: Improvement Statistics**
   - Menunjukkan % improvement dari epoch 1 ke last epoch
   - 3 metrics: Train Loss, Val Loss, Val CER
   - Semakin besar % = semakin bagus training

5. **Bottom-Right: Training Statistics**
   - Best epoch untuk setiap metric
   - Final values
   - Total loss drop

---

### 3️⃣ test_results_analysis.png (Test Evaluation)

**Generated by:** `analyze_results.py`

**Isi (4 subplots):**

1. **Top-Left: CER Distribution**
   - Histogram dari semua test sample CER
   - Red line = Mean CER
   - Green line = Median CER
   - Shows: bentuk distribusi, outliers

2. **Top-Right: CER by Gender**
   - Box plot perbandingan male vs female
   - Shows: performance difference per gender
   - Median (orange line), quartiles (box), outliers (points)

3. **Bottom-Left: CER by Subject (Top 10)**
   - Bar chart subject mana yang paling sulit
   - Horizontal bar = subject names
   - Height = average CER
   - Helps identify problematic subjects

4. **Bottom-Right: Test Results Summary**
   - Statistics tekstual:
     - Total samples
     - CER mean, median, std, min, max
     - Categories: Perfect/Excellent/Good/Fair
     - Subject count, Gender distribution

## 🎯 Interpretasi Hasil

### Loss Values
```
Excellent:  Loss < 0.1
Good:       Loss 0.1 - 0.5
Fair:       Loss 0.5 - 1.0
Poor:       Loss > 1.0
```

### CER Values (Character Error Rate)
```
Perfect:    CER = 0.0        (100% accuracy)
Excellent:  CER < 0.1   (90%+ accuracy)
Very Good:  CER < 0.2   (80%+ accuracy)
Good:       CER < 0.4   (60%+ accuracy)
Fair:       CER < 0.6   (40%+ accuracy)
Poor:       CER ≥ 0.6   (<40% accuracy)
```

### Red Flags 🚩

| Indikator | Problem | Solution |
|-----------|---------|----------|
| Train loss ⬇ Val loss ⬆| Overfitting | Increase regularization, reduce epochs |
| Both losses high ⬆| Underfitting | Increase model capacity, more epochs |
| Loss plateaus | Stuck convergence | Lower learning rate, warm restart |
| High variance | Noisy data | More epochs, larger batch size |
| Very high CER | Model not learning | Check data, reduce learning rate |

## 📊 Contoh Interpretasi

### Skenario A: Ideal Training ✅
```
Plots menunjukkan:
- Train loss: 2.5 → 0.3 (smooth downward)
- Val loss: 2.4 → 0.35 (smooth downward, sedikit lag)
- Val CER: 0.9 → 0.08 (significant improvement)
- Improvement stats: >80% untuk semua metrics
- Test CER distribution: mean 0.12, terfokus

Conclusion: ✓ Good training, ready for deployment
```

### Skenario B: Overfitting ⚠️
```
Plots menunjukkan:
- Train loss: 2.5 → 0.05 (very steep, agresif)
- Val loss: 2.4 → 0.6 (increases after epoch 15)
- Val CER gaps besar antara train dan val

Conclusion: ✗ Model overfitting, reduce learning rate
            atau gunakan lebih awal stopping (early stopping)
```

### Skenario C: Underfitting ❌
```
Plots menunjukkan:
- Train loss: 2.5 → 1.5 (plateaus early)
- Val loss: 2.4 → 1.6 (tidak banyak improvement)
- Val CER: tetap tinggi 0.6+

Conclusion: ✗ Model underfitting, increase model size
            atau lower learning rate lebih agresif
```

## 🔍 Advanced Analysis

### Debugging dengan Visualisasi

**Jika Val CER tinggi despite Low Loss:**
→ Check data quality, ensure targets align with features

**Jika Training oscillates (noisy):**
→ Reduce batch size atau increase learning rate

**Jika CER distributions very wide:**
→ Some subjects easier than others, consider ensemble

    h = json.load(f)

    'epoch': range(1, len(h['train_loss']) + 1),
    'train_loss': h['train_loss'],
    'val_loss': h['val_loss'],
    'val_cer': h['val_cer']
})

df.to_csv('src/pipelines/training/training_history.csv', index=False)
"
